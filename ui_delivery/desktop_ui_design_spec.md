# SMS Bridge 桌面端设计规范

## 1. 设计目标
- 提升视觉现代感与专业可信度
- 降低首次配置与日常操作学习成本
- 强化状态反馈，减少误操作不确定性
- 保证浅色/深色双主题一致体验

## 2. 信息架构
- 顶部：产品标题、说明、主题切换、托盘入口
- 状态卡：运行状态、关键连接信息
- 配置卡：网络与安全参数输入
- 操作卡：保存/启动/停止/隐藏高频动作
- 日志卡：运行反馈与排障信息

## 3. 视觉语言
- 布局：卡片化 + 圆角 + 统一间距（8/12/16）
- 色彩：
  - 浅色背景 `#F3F6FB`，卡片 `#FFFFFF`
  - 深色背景 `#0B1220`，卡片 `#111C33`
  - 主色 `#3B82F6`（浅）/`#60A5FA`（深）
  - 成功 `#10B981`，告警 `#EF4444`
- 字体：
  - 标题：Segoe UI 18 Bold
  - 卡片标题：Segoe UI 12 Bold
  - 正文：Segoe UI 10
  - 日志：Consolas 10
- 图标：托盘图标采用圆角块 + 气泡识别形态

## 4. 交互规范
- 关闭窗口：隐藏至系统托盘，不中断服务
- 托盘菜单：显示窗口 / 启停服务 / 切换主题 / 退出程序
- 主题切换：即时生效，持久化保存
- 状态反馈：成功/失败统一更新状态条和颜色

## 5. 动效规范
- 主题切换：0.25~0.3s 颜色过渡
- 状态变化：状态卡边框脉冲动画
- 卡片交互：悬停轻微抬升（原型）

## 6. 响应式规范
- >=980px：配置卡与操作卡双列
- <980px：双卡纵向堆叠
- 最小窗口：820 x 620

## 7. 组件库
- 主题控制：`ThemeController`
- 设计令牌：`ThemePalette`, `LIGHT_THEME`, `DARK_THEME`
- 容器组件：`Card`
- 状态组件：`StatusBadge`
- 组件代码文件：`sms_ui_components.py`

## 8. 可访问性
- 主文字与背景保持高对比度
- 操作按钮尺寸不小于 40px 高
- 关键状态提供文字+颜色双通道表达

## 9. 实现映射
- GUI 实现文件：`sms2clipboard_gui.py`
- 组件库：`sms_ui_components.py`
- 设计稿：`ui_delivery/desktop_ui_design_mock.html`
- 交互原型：`ui_delivery/desktop_ui_interaction_prototype.html`
